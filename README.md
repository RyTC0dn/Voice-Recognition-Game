# Voice-Recognition-Game
This is a college project of developing a 3D, obstacle course-style game with the intention to have the control inputs be entirely through voice-recognition 

# Experience Design Proposal
Ryan Thompson 



## Preface
“Developing a simplistic 2D obstacle course game entirely controlled through voice recognition input.”

## Design Considerations
### Emotions
The primary emotional goals of this experience are: 
- Enjoyment
- Empowerment

**Enjoyment** mainly from the expressive and sometimes unpredictable nature of voice-based controls. Because vocal input can vary in tone, clarity, and timing, the gameplay may produce humorous or unexpected moments. These moments can encourage a playful, welcoming atmosphere where they get to experience the chaotic interactions that come through speech controls.

**Empowerment** comes from the player’s ability to directly command the character using their own voice. Instead of relying on traditional inputs such as buttons or joysticks, the player issues verbal commands directly to the controller, which translates to movement within the game. This is to inspire that sense of agency from the player, where they will feel more responsible for directing their character’s actions.

## Development Plan
List of skills & materials required: 
### Skills
UI Design - Visual feedback to the player
Unity voice recognition input - main control mapping for player
Frequency detection - managing the audio frequency detection with the microphone input
### Materials
PC x 1
USB Microphone (or other external microphones that can connect to PCs) x 1

### Rough Development Plan
Breaking down the 21 hours: 
Research (3-4 hours) - gathering as much research research and reference material as possible, to start brainstorming ideas on how to implement the voice-recognition system. 
Core Input development (6-7 hours) - developing the coding scripts for the base structure of the voice-recognition control system. 
Level Implementation (3-5 hours) - putting together the actual obstacle course level of the game, since the game will be within a 2D environment, I will be using top-down asset packs to develop the level. 
Polish (3-5 hours) -  the final steps of the development cycle is to polish the design and finalize any visual design (UI) components required for the experience.
