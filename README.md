# Voice-Recognition-Game
This is a college project of developing a 3D, obstacle course-style game with the intention to have the control inputs be entirely through voice-recognition 

# Experience Design Proposal
Ryan Thompson 



## Preface
“Developing a simplistic 2D obstacle course game entirely controlled through voice recognition input.”

## Design Considerations
### Emotions
The primary emotional goals of this experience are: 
Tension
Empowerment
Tension, as it emerges from required controlled movement using vocal inputs. Unlike pressing a button, speaking introduces variability and unpredictability, creating moment-to-moment pressure while navigating to the end of the obstacle course.
Empowerment stemming from direct vocal agency. The player’s voice becomes the sole driver of action, reinforcing engagement within the experience.


### Experience
This project aims to deliver a short, focused, and challenging experience that supports people being able to walk up and play. A player should be able to approach the game and understand the fundamental interaction within moments. 

Essential elements required to achieve this experience are, first off, clear visual feedback from the in-game UI. On top of that, ensure that the controller's mechanical complexity remains low so it doesn’t confuse people when they come up to play it.
 
The main way that the design would come together to reach the essence of the sought experience is to: 
Controls - The main controller for the game will be a standard USB microphone attached to a laptop. Minimal number of controls that need to be learned by the player to play the game. The main inputs for the game are voice commands for the character to move left, right, forward, backward, and jump. 
Visuals - To best provide feedback to the player would be through visual cues within the UI to let you know which of the commands have been received, which also leaves room for audio cues as well. 
Venue
There are mainly two kinds of venues where this game can be experienced; the first being the digital-type venue, online on your pc. The other possible venue would be in a more social environment, such as a classroom or workshop. 

The reason for the digital venue, tied back to my experience goal, is to allow everyone to play the game, where those who may have mobility issues can stay home and enjoy the game, but also allows fewer disturbances with the voice-recognition controls. With the social venues, it helps that it is a more controlled environment where there aren’t any erratic noises that can be picked up by the microphone from environments such as conventions.

## Development Plan
List of skills & materials required: 
### Skills
UI Design - Visual feedback to the player
Unity voice recognition input - main control mapping for player
Frequency detection - managing the audio frequency detection with the microphone input
### Materials
PC x 1
USB Microphone (or other external microphones that can connect to PCs) x 1

### Rough Development Plan
Breaking down the 21 hours: 
Research (3-4 hours) - gathering as much research research and reference material as possible, to start brainstorming ideas on how to implement the voice-recognition system. 
Core Input development (6-7 hours) - developing the coding scripts for the base structure of the voice-recognition control system. 
Level Implementation (3-5 hours) - putting together the actual obstacle course level of the game, since the game will be within a 2D environment, I will be using top-down asset packs to develop the level. 
Polish (3-5 hours) -  the final steps of the development cycle is to polish the design and finalize any visual design (UI) components required for the experience.
